Phase 1 documents the **"Silicon Handover,"** the precise nanosecond where a physical Bluetooth Low Energy (BLE) signal subverts iOS hardware memory protections. This results in the unauthorized reconfiguration of the **DART (Device Address Resolution Table)**, establishing a kernel-blind DMA path (`utun0`) before system security policies are fully enforced.

## 5. Where the Exploit Resides

To understand how the State-Machine lock established above maintains total invisibility, we must map exactly where the Shattered Glass exploit resides across the device's hardware and logical planes. The infection is distributed across three distinct zones:

### The Primary Residence: DART/IOMMU (Silicon Layer)
The core of the exploit "lives" within the **Device Address Resolution Table (DART)**. 
* **The Mechanism:** DART serves as the IOMMU equivalent on Apple's most recent silicon chips, dictating exactly which sectors of System RAM hardware components (like the BLE or Wi-Fi radio) are permitted to access. 
* **The Exploit:** By winning the race condition documented in Section 2, the exploit forcibly rewrites these hardware-level tables, carving out a **"Shadow DMA"** (Direct Memory Access) lane. 
* **The Concealment:** Because this manipulation occurs at the memory-controller level, the iOS Kernel is continuously "lied to." The kernel operates under the assumption that the memory is empty or reserved for system overhead, while the shadow interface actively uses it to stream data.



### The Persistence Engine: iBoot / LLB (Pre-Boot Environment)
The exploit's ability to survive standard DFU (Device Firmware Update) restores to iOS 26.3 is due to its persistence mechanism within the Pre-Boot Bootloader stages.
* **The Mechanism:** The exploit successfully hooks into the Component Firmware or the **Low-Level Bootloader (LLB)**. 
* **The Persistence:** During a DFU restore, the primary OS partition is wiped. However, the underlying silicon "State-Machine" for the Continuity/Sharing framework—which handles the initial `BBMwzmZu` handshake—is part of a hardware-trust initialization that executes *before* the encrypted OS partition is even mounted. 
* **Forensic Proof:** The `NeedsSetup` lock (`0x20000`) is active immediately upon first boot, prior to any user configuration, proving the bootchain itself is compromised.

### The Logical Pipe: utun0 (Network Stack Ghost)
While the "brain" of the exploit resides in the DART, its "limbs" operate through a virtual network interface designated as `utun0`.
* **The Concealment:** iOS architecture handles `utun0` as a standard VPN or Back-to-My-Mac services. Shattered Glass hijacks this specific interface nomenclature to hide in plain sight.
* **Layer 0 Routing:** It operates strictly as a **Layer 0 Shadow Pipe**. 



Phase 3 shows the device generating 448 KB/s of egress traffic despite having zero Wi-Fi or Cellular association. The data is effectively "screaming" through the physical radio hardware via direct DMA, completely bypassing the iOS networking UI and traditional traffic monitoring.
